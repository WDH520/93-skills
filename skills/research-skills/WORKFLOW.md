# 7阶段文献综述工作流

本工作流适用于任何学科领域的文献综述撰写，支持领域自适应。

---

## 阶段0：领域识别与配置

**目标**：根据研究主题自动识别学科领域并配置相应参数

### 0.1 领域自动检测

#### 检测步骤

```
步骤1: 提取用户输入中的所有关键词
步骤2: 匹配DOMAINS.md中的领域关键词表
步骤3: 计算各领域匹配度 = 匹配词数 / 总词数
步骤4: 选择匹配度最高的领域作为主要领域
步骤5: 如匹配度≥50%，自动继续；如<50%，询问用户确认
```

#### 匹配度计算示例

用户输入："深度学习在医学影像诊断中的应用综述"

| 领域 | 匹配关键词 | 匹配度 | 结果 |
|------|-----------|--------|------|
| 计算机科学/AI | 深度学习、诊断 | 2/8=25% | 主要领域 |
| 生物学/医学 | 医学影像、诊断 | 2/8=25% | 次要领域 |
| 其他领域 | - | 0% | - |

**判定**：主要领域=计算机科学/AI，次要领域=生物学/医学（跨领域主题）

### 0.2 加载领域配置

根据检测到的领域，从 [DOMAINS.md](DOMAINS.md) 加载：
- 核心数据库配置
- 典型评价指标
- 推荐文献结构
- 术语规范

### 0.3 多领域交叉处理

如果涉及多个领域：
1. 确定主要领域（方法/理论导向，匹配度最高）
2. 确定次要领域（应用/验证导向，匹配度次高且>30%）
3. 融合数据库配置（主要+次要领域的数据库）
4. 采用主要领域结构，整合次要领域要素

### 0.4 错误处理与回退策略

#### 领域检测失败处理

当自动领域检测无法确定学科领域时，执行以下回退策略：

**场景1：关键词不匹配任何领域**
- 处理：使用通用配置（WebSearch为主）
- 提示："根据您的主题，我无法自动识别具体学科领域。我将使用通用配置进行文献检索。请确认这是否符合您的需求？"

**场景2：多领域冲突（置信度相近）**
- 处理：列出候选领域，请用户选择
- 提示："您的主题可能涉及多个领域：[领域1]、[领域2]。请告诉我哪个是主要研究领域？"

**场景3：用户主题过于模糊**
- 处理：请求用户提供更具体的研究方向
- 提示："您提供的主题较为宽泛。能否提供更多关键词或具体的研究方向？"

#### 常见错误处理

| 错误场景 | 处理策略 | 用户提示 |
|---------|---------|---------|
| 关键词不匹配任何领域 | 使用通用配置，询问用户确认 | "我将使用通用配置，是否继续？" |
| 多领域冲突（置信度相近） | 列出候选领域，请用户选择 | "请选择主要研究领域：[列表]" |
| 用户主题过于模糊 | 请求用户提供更具体的研究方向 | "能否提供更多关键词？" |
| 数据库检索失败 | 切换到备用数据库，记录失败原因 | "[数据库]检索失败，使用[备用]替代" |
| 文献数量不足 | 扩展关键词，放宽日期限制，追溯参考文献 | "文献较少，扩展检索范围" |
| WebSearch返回结果过少 | 调整搜索词，使用同义词扩展 | "调整搜索策略" |

## 阶段1：项目初始化

**目标**：创建标准化的项目结构

### 1.1 确定项目目录

**规则**：在当前工作目录创建项目文件夹

```
如果用户指定了目录：
  使用用户指定目录
否则：
  使用当前工作目录
```

### 1.2 创建项目文件

在当前工作目录创建以下文件：

```
project_root/
├── CLAUDE.md              # 写作指南和术语规范
├── IMPLEMENTATION_PLAN.md # 分阶段执行计划
├── manuscript_draft.md    # 主文稿
└── figures/               # 图占位符目录（可选）
```

### 1.3 初始化 CLAUDE.md

基于 [TEMPLATES.md](TEMPLATES.md) 中的模板，创建包含以下内容的写作指南：
- 术语标准化表（从DOMAINS.md复制对应领域）
- 缩写规范
- 参考文献来源配置
- 文献分类框架
- 质量检查清单

**创建步骤**：
1. 从DOMAINS.md复制对应领域的术语规范
2. 根据主题定制关键词列表
3. 设置文献数量目标（通常80-120篇）
4. 定义引用格式（GB/T 7714）

### 1.4 初始化 IMPLEMENTATION_PLAN.md

创建7阶段执行计划，包括：
- 项目概述（主题、目标、截止日期）
- 各阶段任务清单（使用TodoWrite跟踪）
- 文献映射表
- 变更日志

### 1.5 初始化 manuscript_draft.md

创建主文稿框架：
- 标题（基于用户输入）
- 各级标题结构（基于领域模板）
- 表格占位符
- 图占位符

## 阶段2：文献收集

**目标**：建立全面的文献库（目标80-120篇）

### 2.1 数据库选择策略

根据领域选择核心数据库：

| 领域 | 主要数据库 | 次要数据库 | 目标数量 |
|------|-----------|-----------|---------|
| 计算机科学/AI | ArXiv, IEEE Xplore | ACM, DBLP | 80-100篇 |
| 生物学/医学 | PubMed, PMC | Embase, Cochrane | 60-80篇 |
| 社会科学 | JSTOR, Web of Science | PsycINFO, ERIC | 60-80篇 |
| 工程技术 | IEEE Xplore, EI | ASME, AIAA | 60-80篇 |
| 经济学 | EconLit, RePEc | SSRN, JSTOR | 60-80篇 |

### 2.2 检索工作流

#### 方式1：WebSearch检索（默认）

**适用场景**：所有学科领域，无需额外配置

**检索策略**：
```
1. 基础搜索：
   - "[主题] review"
   - "[主题] survey"
   - "[主题] 综述"

2. 来源限定搜索：
   - site:arxiv.org "[主题]"
   - site:pubmed.ncbi.nlm.nih.gov "[主题]"
   - site:ieeexplore.ieee.org "[主题]"

3. 文件类型限定：
   - "[主题]" filetype:pdf
   - "[主题]" filetype:doc

4. 时间限定：
   - "[主题]" 2020..2024
```

**执行步骤**：
1. 使用WebSearch执行上述搜索
2. 提取搜索结果中的文献信息（标题、作者、年份、来源）
3. 去重处理
4. 按主题分类

#### 方式2：ArXiv MCP检索（如已配置）

**适用领域**：计算机科学、物理学、数学、定量生物学

**检索策略**：
```
1. 使用 search_papers 工具
2. 关键词组合：[主题] AND (method OR approach OR algorithm)
3. 类别筛选：根据领域选择（cs.CV, cs.LG, cs.AI等）
4. 日期范围：近3-5年
5. 目标数量：50-100篇
```

**未配置时的替代方案**：
- 使用WebSearch搜索 `site:arxiv.org "[主题]"`

#### 方式3：PubMed MCP检索（如已配置）

**适用领域**：医学、生物学、生命科学

**检索策略**：
```
1. 使用 PubMed MCP
2. MeSH术语：使用标准医学主题词
3. 筛选条件：
   - Publication Type: Review, Clinical Study
   - Date: 近5年
4. 目标数量：30-50篇
```

**未配置时的替代方案**：
- 使用WebSearch搜索 `site:pubmed.ncbi.nlm.nih.gov "[主题]"`

### 2.3 文献组织

创建文献矩阵：

```markdown
| 类别 | 子类别 | 关键文献 | 数量 | 来源 |
|------|--------|----------|------|------|
| 方法 | 传统方法 | [refs] | N | ArXiv |
| 方法 | 深度学习方法 | [refs] | N | ArXiv |
| 应用 | 临床验证 | [refs] | N | PubMed |
| 数据 | 公开数据集 | [refs] | N | Mixed |
```

### 2.4 质量筛选

**纳入标准**：
- [ ] 同行评审或高影响力预印本
- [ ] 与主题直接相关
- [ ] 近5年文献（经典文献除外）
- [ ] 可获取全文或详细摘要

**排除标准**：
- [ ] 与主题无关
- [ ] 质量过低或方法不明确
- [ ] 重复发表

### 2.5 错误处理

| 问题 | 处理策略 |
|------|---------|
| 检索结果过少(<20篇) | 扩展关键词、放宽日期限制、追溯参考文献 |
| 检索结果过多(>200篇) | 增加限定词、缩小日期范围、限定高质量期刊 |
| 结果质量低 | 增加"review"、"survey"限定词，限定来源 |
| 重复文献多 | 使用去重工具，按标题和作者去重 |

## 阶段3：大纲制定

**目标**：确定论文结构和内容分配

### 3.1 选择结构模板

根据领域从 [DOMAINS.md](DOMAINS.md) 选择适当模板：

- **计算机科学**：引言→背景→方法分类→实验对比→应用→挑战→结论
- **医学/生物学**：引言→方法→结果→讨论→结论（PRISMA风格）
- **社会科学**：引言→理论框架→文献综述→方法论→发现→讨论→结论
- **工程技术**：引言→理论基础→技术方法→性能评估→应用→趋势→结论

### 3.2 文献映射

将收集的文献分配到各章节：

```markdown
| 章节 | 关键文献 | 主要内容 |
|------|----------|----------|
| 3.1 [类别1] | [Paper A], [Paper B] | [要点] |
| 3.2 [类别2] | [Paper C], [Paper D] | [要点] |
```

### 3.3 规划表格和图表

确定必需的表格和图表：

**必备表格**（根据领域调整）：
- 表1：数据来源/数据集
- 表2：方法/理论对比
- 表3：应用/产品/政策

**图占位符**：
- 图1：研究时间线/发展历程
- 图2：方法分类体系
- 图3：性能趋势/结果对比

## 阶段4：章节撰写

**目标**：完成各章节初稿

### 4.1 撰写顺序

推荐撰写顺序：
1. 方法论/数据来源（第2章）- 最熟悉的内容
2. 主体章节（第3章）- 核心内容
3. 引言（第1章）- 基于已完成内容撰写
4. 应用/发现（第4-5章）- 基于主体章节
5. 讨论（第6章）- 综合分析
6. 结论（第7章）- 总结全文

### 4.2 段落写作规范

每个段落应包含：
1. **主题句**：明确主要论点
2. **支持证据**：引用+数据
3. **分析**：批判性评价
4. **过渡**：连接下一段

### 4.3 方法描述模板

```markdown
### 3.X [方法类别]

[1-2段引言，说明动机和背景]

**[方法名称]：** [作者]等[ref]提出了[方法]，其创新点包括：
- [关键特征1]
- [关键特征2]
在[数据/样本]上达到[结果指标]。

**局限性：** 尽管有优势，[类别]方法面临：
(1) [局限1]；
(2) [局限2]。
```

### 4.4 使用 TodoWrite 跟踪进度

为每个章节创建任务：
```markdown
- [ ] 3.1 [类别1] - 撰写中
- [ ] 3.2 [类别2] - 待开始
- [ ] 表2 方法对比 - 待开始
```

## 阶段6：质量保障

**目标**：确保综述质量符合学术标准

### 6.1 结构完整性检查

- [ ] 标题页和摘要
- [ ] 3-5个要点（Key Points）
- [ ] 引言（背景、挑战、贡献）
- [ ] 方法论章节
- [ ] 主要发现章节
- [ ] 讨论（局限性、未来方向）
- [ ] 结论
- [ ] 参考文献（80-120篇）

### 6.2 内容质量检查

**引用完整性**：
- [ ] 每个主要论点都有引用支持
- [ ] 引用格式统一（GB/T 7714）
- [ ] 近5年文献占比≥60%
- [ ] 包含经典/奠基性文献

**内容深度**：
- [ ] 每个方法类别都有局限性讨论
- [ ] 包含批判性分析（不仅是描述）
- [ ] 识别研究空白
- [ ] 提出未来方向

### 6.3 写作规范检查

**语言风格**：
- [ ] 使用谨慎措辞（"可能"、"表明"、"似乎"）
- [ ] 避免绝对性断言
- [ ] 术语使用一致
- [ ] 段落间过渡自然

**学术规范**：
- [ ] 无抄袭
- [ ] 正确引用
- [ ] 数据准确

### 6.4 领域特定检查

根据领域进行专项检查：

**计算机科学/AI**：
- [ ] 实验设置描述清晰
- [ ] 评价指标定义明确
- [ ] 性能数据可复现

**医学/生物学**：
- [ ] 遵循PRISMA指南（如适用）
- [ ] 临床意义明确
- [ ] 伦理考虑提及

**社会科学**：
- [ ] 理论框架清晰
- [ ] 方法论透明
- [ ] 样本特征描述完整

## 阶段7：增量更新

**目标**：持续更新综述内容

### 7.1 更新触发条件

- 新重要文献发表
- 用户要求更新特定部分
- 新方法/理论出现
- 新数据集发布

### 7.2 更新流程

1. **评估新文献**
   - 与现有内容的关联性
   - 对现有结论的影响
   - 是否需要新增章节

2. **确定插入位置**
   - 更新现有章节
   - 添加新小节
   - 创建全新章节

3. **执行更新**
   - 更新 CLAUDE.md 文献来源
   - 更新 IMPLEMENTATION_PLAN.md
   - 修改 manuscript_draft.md
   - 更新表格和图表

4. **版本控制**
   ```markdown
   ## 变更日志
   ### [日期] - v1.1
   - 新增第3.X节 [新类别]
   - 更新表2，增加N个新方法
   - 新增参考文献 #XX-#YY
   ```

## 常见问题处理

### Q1: 如何处理跨领域主题？

A: 
1. 识别主要领域（方法/理论导向）
2. 识别次要领域（应用/验证导向）
3. 采用主要领域的结构模板
4. 整合次要领域的评价指标
5. 融合两个领域的文献来源

### Q2: 文献数量不足怎么办？

A:
1. 扩展关键词（同义词、相关概念）
2. 放宽日期限制（纳入稍早期文献）
3. 检查相关领域数据库
4. 追溯高被引文献的参考文献
5. 使用更宽泛的搜索词

### Q3: 如何处理相互矛盾的文献结论？

A:
1. 客观呈现各方观点
2. 分析矛盾原因（方法差异、样本差异、条件差异）
3. 指出需要进一步研究的问题
4. 使用谨慎措辞表达不确定性

### Q4: 如何确保领域术语准确？

A:
1. 查阅领域权威综述的术语使用
2. 参考 [DOMAINS.md](DOMAINS.md) 中的术语规范
3. 在 CLAUDE.md 中建立术语表
4. 全文保持术语一致

### Q5: MCP工具未配置怎么办？

A:
1. 使用WebSearch作为替代方案
2. 使用site:限定符搜索特定数据库
3. 使用filetype:pdf获取全文
4. 考虑配置MCP工具以获得更好体验
