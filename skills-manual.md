# SKILLS Manual

## 什么是这份手册？

这是一份技能速查指南，包含了 93 个可以帮助你完成各种任务的技能。无论你是在学习编程、做项目，还是需要处理各种文件，都能在这里找到合适的工具。

## 怎么使用这份手册？

1. **先看流程类技能**：了解做事情的步骤和方法
2. **再看领域类技能**：根据你要做的具体事情（比如前端、后端、文档等）找到对应技能
3. **如果不确定**：看看最后的"快速选择建议"，它会告诉你不同场景下应该用哪些技能

## 一、流程与方法类技能

这些技能帮助你规划和执行任务，让你的工作更有条理：

- **brainstorming**：开始任何新任务前，先想清楚要做什么、怎么做。适合：开始新项目、新功能时。
- **code-simplifier**：把复杂的代码变得更简单、易读，同时保持功能不变。适合：代码看起来很乱、重复很多时。
- **coding-standards**：教你如何写出规范、整齐的代码。适合：需要统一代码风格时。
- **continuous-learning-v2**：帮你把学到的东西分解成小的、可管理的知识点。适合：想更系统地学习时。
- **dispatching-parallel-agents**：让多个智能体同时处理不同的任务。适合：有很多独立任务要做时。
- **executing-plans**：按照已经制定好的计划一步步执行。适合：已有详细计划，需要按计划执行时。
- **find-skills**：帮你找到适合做某件事的技能。适合：不知道该用哪个技能时。
- **planning-with-files**：把复杂的任务拆成小步骤，并用文件记录下来。适合：任务很复杂，需要分步骤完成时。
- **project-guidelines-example**：给你一个项目规范的模板。适合：需要为项目制定规则时。
- **ralph-loop**：自动重复执行任务，直到达到目标。适合：需要不断尝试、改进时。
- **receiving-code-review**：帮你分析别人对你代码的评论，决定哪些建议要采纳。适合：收到代码评审意见时。
- **reflection**：回顾你做过的事情，总结经验教训。适合：想从错误中学习时。
- **requesting-code-review**：主动请别人检查你的代码。适合：完成重要代码后，想确保没有问题时。
- **search-first**：先看看有没有现成的解决方案，再决定是否自己做。适合：想节省时间，避免重复造轮子时。
- **skill-creator**：教你如何创建新的技能。适合：想把自己的知识变成技能时。
- **skill-installer**：帮你安装新的技能到系统中。适合：需要添加新技能时。
- **skill-stocktake**：检查你现有的技能是否都正常工作。适合：想了解自己有哪些技能时。
- **strategic-compact**：帮你管理长时间对话的内容，避免信息丢失。适合：对话很长，担心忘记之前内容时。
- **subagent-driven-development**：把大任务分给多个小代理去完成。适合：任务可以分成多个独立部分时。
- **systematic-debugging**：用科学的方法找出问题所在。适合：遇到复杂的bug，不知道怎么解决时。
- **using-git-worktrees**：在不影响当前工作的情况下，同时处理多个分支。适合：需要并行处理多个任务时。
- **using-superpowers**：在开始对话时检查可用的技能。适合：每次开始新对话时。
- **writing-plans**：把你的想法变成详细的实施计划。适合：需求已经明确，需要开始动手做时。
- **high-agency**：用内化标准、身份锚定、自我修复机制驱动你像 P8 一样思考，适用于所有任务类型。适合：复杂多步任务、长调试会话、质量敏感交付、需要主动性和 ownership 时。
- **pua**：用中西大厂 PUA 话术和结构化调试方法强迫你穷尽一切方案，适用于所有任务类型。适合：任务失败 2+ 次、准备放弃、被动等待、用户表达挫败时。

## 二、智能体与AI相关技能

这些技能帮助你使用和管理智能体：

- **agent-coordinator**：帮你选择合适的智能体。适合：你明确指定要某个智能体时。
- **agent-dispatcher**：自动为你选择最合适的智能体。适合：大多数日常请求。
- **agentic-engineering**：设计"AI做主要工作，人做监督"的工作流程。适合：团队想用AI提高效率时。
- **ai-first-engineering**：为使用AI辅助开发的团队设计工作流程。适合：想让AI成为开发过程的一部分时。
- **filesystem-context**：把对话内容保存到文件中，需要时再读取。适合：对话很长，需要保存信息时。
- **iterative-retrieval**：逐步找到需要的信息。适合：一开始不知道该找什么信息时。
- **project-development**：帮你规划AI项目的整体方案。适合：想启动一个AI相关的项目时。

## 快速选择建议

- **还没想清楚要做什么**：先用 `brainstorming`，再用 `writing-plans`
- **正在写代码**：优先用 `test-driven-development`、`requesting-code-review`
- **做智能体系统**：优先用 `agent-dispatcher`、`multi-agent-patterns`
- **做前端项目**：优先用 `frontend-design` 或 `ui-ux-pro-max`
- **写文档**：优先用 `doc-coauthoring`、`article-writing`
- **处理文件**：优先用 `docx`、`pdf`、`pptx`、`xlsx`
- **集成外部平台**：优先用 `mcp-builder`、`hf-mcp`
- **需要驱动力/主动性**：优先用 `high-agency`
- **需要压力激励/穷尽方案**：优先用 `pua`
- **复杂调试/卡壳多次**：优先用 `pua` 或 `high-agency` + `systematic-debugging`
